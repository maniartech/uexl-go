# Agents.md

## Development guidelines

### Must follow these principles for developing any code

- Do not over-engineer
- Write clean, readable, and maintainable code
- KISS (Keep It Simple, Stupid) - Never overcomplicate solutions. When in doubt, ask and suggest simpler yet effective approaches.
- YAGNI (You Aren't Gonna Need It) - Avoid adding features or complexity until they are actually needed.
- DRY (Don't Repeat Yourself)
- SRP (Single Responsibility Principle)

### Code Quality

- Must be militry-grade robust and secure and thread-safe
- Must be able to handle edge cases and unexpected inputs gracefully
- Must be able to handle high concurrency and load without crashing or degrading performance
- Must be well-documented with clear comments and documentation for public APIs
- Must follow Go idioms and best practices as outlined in "Effective Go" and "Go Proverbs"
- Must be super efficient and optimized for performance, with minimal allocations and low latency
- Must be designed with scalability in mind, allowing for easy extension and maintenance in the future
- Must be future-proof and adaptable to changing requirements and technologies, with a focus on modularity and flexibility

#### Code Quality Assurance
If you are unsure about any of the above principles or guidelines, please ask for clarification or guidance before proceeding with development. It is better to ask questions and ensure that you are on the right track than to make assumptions and potentially create code that does not meet the required standards.
